# STORAGE ARCHITECTURE

## 저장 전략

MVP는 로그인 없이 사용 가능해야 하며, 사용자 상태는 가능한 한 로컬 기기에 저장한다.

## 저장 목표

- 서버 비용 최소화
- 가입 장벽 제거
- 오프라인에서도 기본 학습 가능
- 개인 학습 상태를 빠르게 불러오기

## 로컬 저장 대상

- 최근 레벨 선택
- 오답 목록
- 학습 이력
- Daily Goal 상태
- streak
- TTS 관련 설정
- 광고 관련 간단한 로컬 플래그

## 예상 저장 계층

### 1. Static bundled data

- 앱에 포함되는 단어 데이터
- 레벨별 JSON 또는 로컬 DB

### 2. Local app storage

- AsyncStorage 또는 MMKV 같은 키-값 저장소
- 간단한 설정과 상태 저장

### 3. Local structured storage

- SQLite 또는 유사 로컬 DB
- 오답, 학습 이력, 복습 큐 저장

## 서버 사용 방침

- MVP에서 필수 서버 없음
- 사용자 계정 없음
- 기기 간 동기화 없음

## 데이터 유실 정책

- 앱 삭제 시 로컬 데이터는 삭제될 수 있음
- 로그인 없이 사용하므로 이를 제품 정책에 명시
